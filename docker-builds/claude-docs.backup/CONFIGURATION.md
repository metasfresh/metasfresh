## Build Arguments

### Common Build Arguments

| Argument | Usage | Example |
|----------|-------|---------|
| `REFNAME` | Version tag for base images | `--build-arg REFNAME=5.175-master.456` |
| `VERSION` | Full version string for metadata | `--build-arg VERSION=5.175.3-master.456` |
| `REGISTRY` | Docker registry prefix | `--build-arg REGISTRY=myregistry.io/` |

## Secrets

### Maven Settings (mounted at build time)

- **Secret ID**: `mvn-settings`
- **Mount point**: `/root/.m2/settings.xml`
- **Usage**: `--secret id=mvn-settings,src=docker-builds/mvn/settings.xml`
- **Required for**: All Maven builds (common, backend, camel, junit, cucumber)

## Configuration Files

### version.info
- **Content**: Single line with version number (e.g., `5.175`)
- **Usage**: Read by CI to construct full version tags
- **Update**: Manually when starting new release

### mvn/settings.xml
```xml
<server>
  <id>github-metasfresh-maven</id>
  <username>${env.GITHUB_ACTOR}</username>
  <password>${env.GITHUB_TOKEN}</password>
</server>
<server>
  <id>github-3rdparty-maven</id>
  <username>metasfresh-service-account</username>
  <password>$METASFRESH_PACKAGES_READ_TOKEN</password>
</server>
```

- **Placeholders**: Mixed syntax - Maven `${env.VAR}` for some, shell `$VAR` for others (envsubst replaces `$VAR` syntax)
- **CI**: `envsubst` substitutes `$METASFRESH_PACKAGES_READ_TOKEN` at build time
- **Local**: Replace token values manually for local builds

### metadata/build-info.properties (Generated by CI)
```properties
build.name=github actions
build.system=runner-name
build.time=2025-01-15T10:30:00
build.user=github-actor
build.version=5.175.3-branch.123
build.number=123
```

- **Creation**: CI init job
- **Injection**: Repackaged into JAR files
- **Usage**: Displayed in application about/info pages

### metadata/git.properties (Generated by CI)
```properties
git.remote.origin.url=https://github.com/metasfresh/metasfresh
git.branch=refs/heads/master
git.commit.id=abc123def456
git.commit.message=Fix bug in order processing
```

- **Creation**: CI init job
- **Injection**: Repackaged into JAR files
- **Usage**: Traceability to source code

### compose/.env
```
mfregistry=metasfresh
mfversion=5.175-local
dbqualifier=preloaded
```

- **mfregistry**: Docker registry organization
- **mfversion**: Image tag to use (local, or CI tag)
- **dbqualifier**: Database variant (empty, preloaded, postcucumber) — compose templates add the hyphen separator

## Local vs CI Builds

### Local Builds

1. **Version Tag**: `local` or `local-{timestamp}`
2. **Settings**: Use `mvn/local-settings.xml` (manually created)
3. **Cache**: Local Docker layer cache only
4. **Metadata**: Minimal (no build-info.properties)
5. **Registry**: Images stay local, not pushed

### CI Builds (GitHub Actions)

1. **Version Tag**: `{version}-{branch}.{build-number}`
2. **Settings**: Generated via `envsubst` from secrets
3. **Cache**: `--cache-from` remote images + `--cache-to` inline
4. **Metadata**: Full build-info.properties and git.properties
5. **Registry**: Pushed to Docker Hub with retry logic
